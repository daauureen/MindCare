# MINDCARE — веб-прототип

Платформа психологической поддержки студентов: психологи публикуют тесты, студенты проходят их и сами решают, показывать ли результат специалисту.

Это **прототип на React + Vite**, а не мобильное приложение. Он открывается в браузере и нужен, чтобы проверить логику продукта и показать её психологам, студентам и вузу до того, как писать Flutter-версию.

---

## Запуск

Нужен [Node.js 18+](https://nodejs.org).

```bash
npm install
npm run dev
```

Откроется `http://localhost:5173`.

Сборка для публикации:

```bash
npm run build     # результат в папке dist/
npm run preview   # посмотреть собранную версию локально
```

Папку `dist` можно выложить на Netlify, Vercel или GitHub Pages — бэкенд для этого не нужен.

## Как пройти весь сценарий за 5 минут

1. **«Я психолог»** → анкета (3 шага) → загрузите диплом (PDF, JPG или PNG до 3 МБ) → отправьте заявку.
2. Введите код подтверждения почты — в прототипе он показан прямо на экране.
3. Увидите экран «проверим в течение 24 часов» с обратным отсчётом. Тесты создавать пока нельзя — так и задумано.
4. Выйдите, войдите администратором → **Заявки** → откройте документ прямо в панели → **Подтвердить**.
5. Вернитесь под психологом → **Создать тест**: метаданные → вопросы с баллами → диапазоны результата → опубликовать.
6. Зарегистрируйте студента, пройдите тест, отправьте результат.
7. Под психологом откройте результат: ответы по вопросам, сумма баллов, приватная заметка, статус.

## Что работает

- Три роли: студент, психолог, администратор
- Подтверждение почты кодом при регистрации
- Загрузка документов (PDF, JPG, PNG) и просмотр их администратором
- Верификация психолога: подтвердить, запросить документы, отклонить с причиной; повторная подача
- Конструктор тестов: вопросы, баллы за варианты, диапазоны результата, черновик и публикация с валидацией
- Прохождение теста, подсчёт баллов, экран результата со шкалой всех диапазонов
- Явное согласие на отправку результата психологу и отзыв согласия
- Кабинет психолога: список результатов, ответы студента, заметка, статусы
- AI-чат с кризис-детектором и экраном экстренной помощи
- Админ: блокировка пользователей, скрытие тестов, статистика, журнал действий

## Чего нет (и почему)

- **Безопасности.** Пароли хранятся в открытом виде, ролевые проверки только на клиенте. Так делать в проде нельзя.
- **Писем.** Код подтверждения показывается на экране, потому что отправка почты требует сервера.
- **Пуш-уведомлений.** Уведомления складываются в базу, но никуда не отправляются.

Данные теперь хранятся в **Supabase** (PostgreSQL + Storage): пользователи, тесты, попытки, документы и уведомления живут в облачной базе, а не в `localStorage`. Демо-данные при первом запуске уже лежат в базе — регистрация и вход работают сразу.

## AI-чат

По умолчанию чат отвечает подсказкой, что ключ не задан. Чтобы включить его локально:

```bash
cp .env.example .env
# впишите VITE_ANTHROPIC_API_KEY=sk-ant-...
npm run dev
```

Ключ попадает в код страницы и виден любому, кто откроет DevTools, поэтому так можно делать **только на своём компьютере**. В рабочей версии запрос к модели идёт через ваш бэкенд, а ключ остаётся на сервере.
